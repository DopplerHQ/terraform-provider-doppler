# doppler_trusted_ips (Resource)

Manage the trusted IP allowlist for a Doppler config. Only requests from the specified IP ranges will be permitted to access the config's secrets via the API.

Requires a **Team or Enterprise plan**. Collaborators and other project roles with the `enclave_project_config_trusted_ips` permission have permission to manage trusted ips for a config.

## Example Usage

```terraform
resource "doppler_trusted_ips" "backend_prd" {
  project = "backend"
  config  = "prd"
  trusted_ips = [
    "203.0.113.0/24",
    "198.51.100.5/32",
    "2001:db8::1/128"
  ]
}

resource "doppler_trusted_ips" "backend_stg" {
  project = "backend"
  config  = "stg"
  trusted_ips = ["203.0.113.0/24"]
}

resource "doppler_trusted_ips" "backend_dev" {
  project = "backend"
  config  = "dev"
  trusted_ips = ["0.0.0.0/0"]
}
```

### Required

- `project` (String) The name of the Doppler project where the config is located
- `trusted_ips` (Set of String) List of trusted IP ranges in CIDR notation (e.g. 203.0.113.0/24, 1.2.3.4/32). Use 0.0.0.0/0 to allow all traffic.

- `id` (String) The ID of this resource.

## Import

An existing trusted IP configuration can be imported using `<project>.<config>`:

```shell
terraform import doppler_trusted_ips.backend_prd backend.prd
```

## Notes

- **Warning:** Reconfiguring trusted IPs could lock you out of the config if the current client IP is not included in the trusted list.
- The default state for any config is `["0.0.0.0/0"]` (allow all traffic). Deleting this resource restores that default.
- Trusted IPs restrict API access only. They do not affect Dashboard logins.
- IP addresses must be specified in CIDR notation. Use `/32` for an exact IPv4 address and `/128` for an exact IPv6 address (e.g. `1.2.3.4/32` or `2001:db8::1/128`).
